## Problem Statement

TrendLens MVP is built: it crawls Reddit, Geeknews, Anthropic Blog, ranks articles by
popularity, translates to Korean, and shows a ranked feed. But the current product is
a feed — it doesn't have an opinion about what matters *to this specific user*.

The gap: there's a "daily summary" (top 10 articles → Gemini → 3-5 line recap) but it
speaks to no one in particular. The user wants to open TrendLens every morning as a
habit. That requires the briefing to feel written for them, not for a generic reader.

---

## What Makes This Cool

The "당신에게" (For You) layer. Every article card gets a 1-2 line note written to the
user's specific knowledge level: "given that you already understand MoE, here's why
this paper is interesting — and where it contradicts what you read last week."

This is NOT content filtering (showing fewer articles). It's presentation personalization
— same content, explained differently depending on who's reading.

The landscape insight: most "personalized news" apps try to solve selection (which
stories to show). This solves explanation (how to present each story). No ML, no cold
start, no filter bubble. Just a user profile injected into every Gemini prompt.

- Gemini Flash API for all AI features (translation, summary, tooltips)
- No auth system, no accounts (single-user app for now)
- Local-first (deployed to localhost, no cloud deploy yet)

## Premises

1. The core value of TrendLens is saving time on Korean-language AI news — not source
   discovery, but making existing sources digestible. ✅ Confirmed.

2. "Personalization" in this context means context-aware *explanation* (adapt the
   writing to user knowledge level), not algorithmic content filtering. ✅ Confirmed.

3. The fastest path to a daily habit is a morning briefing you'd feel bad missing —
   not more features. ✅ Confirmed.

4. Since there is one user, a static profile (background, expertise areas, what you
   already know) is sufficient — no user account system needed. ✅ Confirmed.

## Landscape Research

Searched: "personalized AI news briefing 2025", "AI newsletter personalization startup
2025", "why personalized news apps fail"

Key findings:
- **ChatGPT Pulse** (Sept 2025) infers interests from conversation history — most
  important product insight of 2025 in this space: removes setup friction entirely
- **Nieman Lab** (June 2025): users want summaries (27%) and translations (24%) most —
  utility beats personalization. Recommendations ranked 21%.
- **Artifact** (Kevin Systrom) shut down 2024 — over-personalization + filter bubbles
- Most "personalized news" apps fail on cold start (need behavioral data to work)

**EUREKA:** Everyone in this space tries to solve "which stories to show" (selection).
Our user's insight is about "how to explain the story" (presentation). Same article,
different framing depending on reader expertise. No ML needed. No cold start.
Ships with a JSON file and a few lines of prompt change in ai_service.py.

## Approaches Considered

### Approach A: Profile-Injected Prompts (Minimal Viable)
- Add `user_profile.json` with background, expertise, context
- Inject into every Gemini prompt — every article summary, daily briefing, tooltip
- Effort: S (2-3 hours) · Risk: Low
- Pro: Ships today. Every existing feature immediately gets better.
- Con: Feed layout unchanged — still feels like a ranked list, not an analyst.

### Approach B: Narrative Morning Briefing (Chosen)
- User profile JSON (as in A)
- Redesign daily briefing: narrative overview written to the user's level
- Add "당신에게" callout to each article card: 1-2 lines explaining why this
  specific article matters given what the user already knows
- New section hierarchy: Narrative Briefing → Today's Picks → Remaining Articles
- Effort: M (4-6 hours) · Risk: Low-Med
- Pro: Transforms TrendLens from feed to analyst. Highest daily habit potential.
- Con: More Gemini prompt engineering required; new frontend section.

## Recommended Approach

**Approach B: Narrative Morning Briefing**

Because the goal is a daily habit — and a feed (even a better-worded one) doesn't
create habit. A briefing that speaks directly to you does.

### Implementation Plan

**1. user_profile.json** — create at `app/config/user_profile.json`

Profile is edited by directly modifying this file. No UI needed for single-user setup.
If file is missing, `load_user_profile()` raises a `FileNotFoundError` with a clear
message: "Create app/config/user_profile.json — see design doc for format."

```json
{
  "name": "ehson",
  "expertise": ["transformer architecture", "LLM pre-training", "FastAPI", "Python"],
  "already_knows": ["MoE basics", "RLHF", "Constitutional AI concept"],
  "interests": ["AI reasoning", "inference optimization", "robotics", "AI startups"],
  "context": "AI/software engineer building AI applications in Korea. Reads Korean."
}
```

**2. Update ai_service.py — inject profile into all prompts**

Add `load_user_profile()` at module level (loaded once at import, not per-request).
Returns a dict. Pass profile into:
- `generate_narrative_briefing()` — renamed from `generate_daily_summary()`
- `translate_and_summarize_article()` — add `why_for_user` field
- `generate_keyword_tooltip()` — calibrate explanation depth to expertise

Concrete before/after for `translate_and_summarize_article()`:
```python
# BEFORE
prompt = f"다음 기사를 한국어로 번역하고 1줄 요약을 작성하세요:\n{article_text}"

# AFTER
profile = load_user_profile()
profile_ctx = (
    f"독자 배경: {', '.join(profile['expertise'])}\n"
    f"이미 아는 것: {', '.join(profile['already_knows'])}\n"
    f"관심사: {', '.join(profile['interests'])}"
)
prompt = f"""{profile_ctx}

다음 기사를 한국어로 번역하고 아래 형식으로 출력하세요:
- 제목 (한국어):
- 요약 (1줄):
- 당신에게 (1-2줄, 독자의 기존 지식과 연결하여 이 기사가 왜 흥미로운지):

기사:
{article_text}"""
```

**3. Rename and rewrite: `generate_narrative_briefing()`**

Rename `generate_daily_summary()` → `generate_narrative_briefing()`. Update the
call site in `main.py` (or wherever `generate_daily_summary` is called).

New prompt format (total response under 120 words):
```
오늘의 흐름: [2-3 sentence narrative about the dominant theme of today's news,
written assuming the user's expertise level]

당신이 이미 아는 것: [1 sentence connecting today's theme to user's prior knowledge]

특히 주목할 기사: [reference to 1-2 specific articles with brief "why"]
```

Note: pass only article titles + summaries (not full content) into this prompt
to stay well within Gemini Flash's context window.

**4. DB schema change**

```sql
ALTER TABLE articles ADD COLUMN why_for_user TEXT;
```

Run once. Existing rows will have `NULL` — frontend shows no callout if `NULL`.
No migration needed (SQLite handles missing column gracefully on SELECT).

`why_for_user` is populated **at crawl time** (batch, alongside translation).
Accept stale values if user profile changes — this is a v1 tradeoff. A "regenerate"
button is a future improvement, not required now.

**5. Frontend changes** — edit `app/static/index.html`

New section structure (replace current layout):
1. `<div class="narrative-briefing">` — narrative overview at top
2. `<div class="picks-section">` — "오늘의 픽 · 당신에게 새로운 내용" (ranks 1-5, full card)
3. `<div class="remaining-section">` — "나머지 인기 기사 · 6-10위" (compact list)

New component per article card in the picks section:
```html
<div class="why-for-you">당신에게: {why_for_user}</div>
```
Styled: left-border 2px solid #888, background #f5f5f2, italic, 12px.
