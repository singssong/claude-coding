# PRD (Product Requirements Document)
## TrendLens - AI 기술 트렌드 뉴스 집약 서비스

**작성일**: 2026-03-26
**버전**: v0.1
**상태**: 기획 확정

---

## 1. 제품 개요

### 1.1 서비스 소개
AI, 데이터센터, 로봇, 스타트업 분야의 해외 기술 기사를 매일 자동 수집·번역·요약하여, 인기도 기반 순위로 제공하는 웹 서비스.

### 1.2 핵심 가치
- 흩어진 기술 커뮤니티(Reddit, Geeknews 등)의 인기 기사를 한 곳에서
- 영어 기사도 한국어로 번역하여 제공
- 기사 내 전문 용어에 마우스를 올리면 바로 설명 확인
- 매일 아침 AI가 오늘의 핵심 이슈를 3~5줄로 요약 제공

---

## 2. 목표 사용자

| 구분 | 설명 |
|------|------|
| 주요 타겟 | AI/기술 트렌드에 관심 있는 국내 직장인, 기획자, 엔지니어 |
| 특성 | 매일 기술 뉴스를 팔로우하고 싶지만, 여러 사이트를 직접 돌아다니기 번거로운 사람 |
| 언어 | 한국어 사용자 (영어 기사 번역 제공) |

---

## 3. 주요 기능 요구사항

### 3.1 뉴스 수집 (크롤링)

| 항목 | 내용 |
|------|------|
| 수집 주기 | 매일 07:00 자동 실행 |
| 기본 수집 소스 | Reddit (r/MachineLearning, r/artificial, r/technology), Geeknews, EO Planet, Nvidia Blog, Anthropic Blog |
| 사용자 추가 소스 | 설정 페이지에서 URL 추가/삭제 가능 |
| 인기도 측정 기준 | 클릭 수 + 댓글 수 (소스별 가중치 적용 가능) |
| 수집 카테고리 | AI, 데이터센터, 로봇, 스타트업 |

### 3.2 메인 피드

| 항목 | 내용 |
|------|------|
| 오늘의 핵심 이슈 | 상단 고정, AI 생성 3~5줄 요약 |
| 뉴스 순위 | 인기도 기반 전체 통합 순위 (1위~N위) |
| 뉴스 카드 구성 | 순위 번호, 제목(한국어 번역), 출처, 요약 1줄, 키워드 태그, 원문 링크 |
| 정렬 기준 | 전체 인기순 (단일 피드) |

### 3.3 키워드 툴팁

| 항목 | 내용 |
|------|------|
| 트리거 | 기사 내 기술 키워드에 마우스 호버 |
| 내용 | AI가 생성한 간결한 개념 설명 (2~3줄) |
| 예시 | "MoE" 호버 → "Mixture of Experts의 약자로, 여러 전문가 모델 중 입력에 맞는 일부만 활성화하는 구조입니다." |

### 3.4 설정 페이지

| 항목 | 내용 |
|------|------|
| 크롤링 소스 관리 | 기본 소스 목록 표시, 사용자 URL 추가/삭제 |
| 수동 크롤링 실행 | "지금 수집" 버튼 |

## 4. 기술 스택

| 구분 | 기술 |
|------|------|
| 백엔드 | Python, FastAPI |
| 데이터베이스 | SQLite (MVP) → PostgreSQL (확장 시) |
| 스케줄러 | APScheduler |
| 크롤링 | BeautifulSoup4, httpx |
| AI | Google Gemini Flash Preview (gemini-2.0-flash-preview) |
| 번역/요약/툴팁 | Gemini API 활용 |
| 프론트엔드 | HTML, CSS, JavaScript (Vanilla) |
| 배포 | 미정 (로컬 우선) |

## 6. 데이터 모델 (초안)

### articles 테이블
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER | PK |
| title_original | TEXT | 원문 제목 |
| title_ko | TEXT | 번역 제목 |
| summary_ko | TEXT | 한국어 요약 |
| url | TEXT | 원문 링크 |
| source | TEXT | 출처 사이트 |
| category | TEXT | AI / 데이터센터 / 로봇 / 스타트업 |
| score | INTEGER | 인기도 점수 |
| comment_count | INTEGER | 댓글 수 |
| view_count | INTEGER | 조회 수 |
| keywords | TEXT | JSON 배열, 키워드 목록 |
| crawled_at | DATETIME | 수집 시각 |

### daily_summary 테이블
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER | PK |
| date | DATE | 날짜 |
| summary | TEXT | 오늘의 핵심 이슈 (AI 생성) |
| created_at | DATETIME | 생성 시각 |

### sources 테이블
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER | PK |
| name | TEXT | 소스 이름 |
| url | TEXT | 소스 URL |
| is_default | BOOLEAN | 기본 소스 여부 |
| is_active | BOOLEAN | 활성화 여부 |

## 7. 비기능 요구사항

| 항목 | 내용 |
|------|------|
| 크롤링 실패 처리 | 소스별 실패 시 해당 소스만 스킵, 나머지 정상 진행 |
| API 비용 최적화 | 키워드 툴팁은 최초 1회 생성 후 DB 캐싱 |
| 확장성 고려 | 유저 테이블 설계 준비 (MVP에서는 미구현) |
| 언어 | UI 및 기사 번역 모두 한국어 |
