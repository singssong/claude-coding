## Affected Pages/Routes
- `GET /` (index.html) — narrative briefing section, picks section, remaining section
- `GET /api/articles` — must include `why_for_user` field in response
- `GET /api/daily-summary` — must return narrative format (v2) after crawl
- `POST /api/articles/{id}/translate` — must not regenerate `why_for_user` if already set

## Key Interactions to Verify
- Page loads with profile pill visible ("브리핑: AI 연구자 프로필 적용 중")
- Top-5 cards show "당신에게:" callout when `why_for_user` is populated
- Cards with `why_for_user=NULL` or empty render cleanly with no callout
- Narrative briefing section renders instead of old plain summary
- Remaining articles (6-10) show compact layout without callout

## Edge Cases

### `load_user_profile()` (tests/test_ai_service.py)
- File exists, valid JSON → returns dict with all expected keys
- File does not exist → raises FileNotFoundError with message containing path hint
- File exists but invalid JSON → raises json.JSONDecodeError (or wrapped ValueError)

### `_build_profile_context(profile)` (tests/test_ai_service.py)
- All fields present → returns string containing name, expertise, already_knows, interests
- Missing `interests` key → does NOT raise KeyError (use `.get()` with default [])
- Empty lists → returns string without crashing (graceful empty values)

### `translate_and_summarize()` fallback (tests/test_ai_service.py — mock Gemini)
- Gemini returns "" → `why_for_user` is None or empty in result (not stored as "")
- JSON parse fails (Gemini returns prose) → result has title_ko, summary_ko, keywords, why_for_user=None

## Critical Paths
- Full crawl → top-5 articles have `why_for_user` populated → page loads with callouts
- Edit `user_profile.json` → within 60 seconds → next Gemini call uses new profile (TTL cache test)
- `ALTER TABLE articles ADD COLUMN why_for_user TEXT` already run → second `init_db()` call does not crash
- `ALTER TABLE daily_summary ADD COLUMN format_version TEXT` already run → second `init_db()` call does not crash
